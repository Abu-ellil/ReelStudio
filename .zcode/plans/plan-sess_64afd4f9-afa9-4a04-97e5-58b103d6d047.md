# خطة إصلاح توزيع المحتوى في المشاهد العمودية

## السبب الجذري
المشاهد تُرسَم على `<canvas>` بإحداثيات يدوية (وليست DOM/CSS). أربع دوال رسم تثبّت المحتوى عند أعلى الشاشة وتترك الأسفل فارغاً، ولا تستخدم نمط التوسيط المركزي الموجود أصلاً في `sceneBarChart` (سطر 1905–1909).

## النمط المرجعي الصحيح (موجود في الكود)
```js
const topY = H*0.18, bottomMargin = H*0.08;
const availH = H - topY - bottomMargin;
const totalH = /* ارتفاع الكتلة */;
const startY = topY + Math.max(0, (availH - totalH)/2);  // توسيط عمودي
```

## التغييرات (كلها في `AI_Reel_Studio_Unified.html`)

### 1) قائمة النقاط — `sceneBullet` (السطر 1475–1476)
استبدال:
```js
const startY = H*0.17;
const gap = Math.min(110, (H-startY-120)/Math.max(items.length,1));
```
بمنطق توسيط المجموعة على نمط `sceneBarChart`:
```js
const topY = H*0.18, bottomMargin = H*0.10;
const availH = H - topY - bottomMargin;
const gap = Math.min(140, availH/Math.max(items.length,1));
const itemH = Math.max(numR*2, lineH);
const totalH = (items.length-1)*gap + itemH;
const startY = topY + Math.max(0, (availH - totalH)/2);
```
(السطر 1487 `const y = startY + i*gap` يبقى كما هو؛ يصبح مرتكزاً على المجموعة المُوسَّطة.)

### 2) قائمة التحقق — `sceneChecklist` (السطر 1864–1865)
نفس التغيير بالضبط (بنية مطابقة). (`const y = startY + i*gap` في السطر 1876 يبقى.)

### 3) المقارنة — `sceneComparison` (السطر 1532)
استبدال `const cardY = H/2 - cardH/2 + H*0.04;` بتوسيط البطاقات داخل المنطقة أسفل العنوان:
```js
const topY = H*0.18, bottomMargin = H*0.08;
const cardY = topY + Math.max(0, ((H - topY - bottomMargin) - cardH)/2);
```

### 4) الرأي — `sceneTestimonial` (الأسطر 1808، 1827، 1842)
حالياً الأفاتار عند `H*0.26` ومربع الاقتباس عند `H*0.52`، مما يترك ~700px فارغة أسفل الشاشة والمجموعة غير متمركزة. سأعيد الحساب بحيث تتمركز الكتلة كاملةً (أفاتار + نجوم + صندوق) عمودياً داخل هامش علوي/سفلي متوازن:
- حساب `totalH` للكتلة من أعلى الأفاتار إلى أسفل صندوق الاقتباس.
- تعريف `blockTop = H/2 - totalH/2` (مع هامش علوي أدنى `H*0.12`).
- اشتقاق `avY`، `starY`، `by` من `blockTop` مع نفس المسافات النسبية الحالية.
هذا يحافظ على الترتيب والتصميم الأفقي كما هو، فقط يُعيد التموضع العمودي.

## النقاط المهمة
- التغييرات رياضية إحداثية فقط داخل دوال الرسم — لا تغيير هيكلي، لا قاعدة بيانات، لا DOM.
- الكسور النسبية لـ `H/W` تعمل تلقائياً مع كل الأشكال (عمودي/مربع/أفقي) كما هو الحال حالياً.
- لا تُمَس بيانات المستخدم أو المشاريع المحفوظة.
- الاختبار: معاينة كل من المشاهد الأربعة (قائمة نقاط، قائمة تحقق، مقارنة، رأي) للتأكد من التوسيط الصحيح وعدم وجود اقتراب من الحواف، خاصةً مع نصوص متعددة الأسطر.

## ملاحظة حول حدّ التوسيط
تقدير `totalH` للعناصر يفترض سطراً واحداً لكل عنصر (تقدير متحفظ). إذا التفّ نص عنصر حول عدة أسطر فإن العنصر قد يمتد قليلاً تحت تقدير المركز، لكنه سيبقى أفضل بكثير من الوضع الحالي (المحتوى في الأعلى). يمكن ضبط ذلك لاحقاً إذا لزم.
