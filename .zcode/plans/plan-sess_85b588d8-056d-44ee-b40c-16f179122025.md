## نظرة عامة على التدفق الجديد

```
عنوان/موضوع  →  لغة + لهجة + نبرة  →  بحث Brave (إن وُجد المفتاح)
       ↓
سكربت من LM Studio  ←  بحث + عنوان + تعليمات اللهجة
       ↓
صندوق معاينة قابل للتعديل  +  [موافق ✅] / [إعادة 🔄]
       ↓ (موافقة)
سكربت → #scriptInput → generateScenes() الموجودة (مع prompt موسّع)
       ↓
المشاهد تُولّد تلقائياً بأنواع مناسبة لكل جزء
```

## الخطوة 2: دالة البحث — `searchWeb(query)`

```js
async function searchWeb(query, count = 5) {
  const key = document.getElementById('braveApiKey').value.trim();
  if (!key) return []; // بدون مفتاح → يعتمد على معرفة النموذج فقط
  const res = await fetch(
    `https://api.search.brave.com/res/v1/web/search?q=${encodeURIComponent(query)}&count=${count}`,
    { headers: { 'X-Subscription-Token': key, 'Accept': 'application/json' } }
  );
  if (!res.ok) throw new Error(`Brave Search HTTP ${res.status}`);
  const data = await res.json();
  return (data.web?.results || []).map(r => ({
    title: r.title, desc: (r.description||'').slice(0,300), url: r.url
  }));
}
```

السلوك الآمن: لو المفتاح غير متاح → ترجع `[]` والتطبيق يكمل بمعرفة النموذج. لو المفتاح خاطئ → تظهر رسالة خطأ لكن المستخدم يقدر يختار "متابعة بدون بحث".

## الخطوة 3: دالة توليد السكربت — `generateScriptFromAI()`

prompt مخصّص لكتابة سكربت فيديو قصير (reel) يأخذ في الاعتبار:
- **العنوان/الموضوع**
- **اللغة** (عربي/إنجليزي)
- **اللهجة** (فصحى/خليجي/مصري/شامي/مغربي) مع تعليمات صريحة لكل لهجة
- **النبرة**
- **نتائج البحث** (إن وُجدت) كسياق معرفي

```js
async function generateScriptFromAI(topic, lang, dialect, tone, research) {
  const model = document.getElementById('modelSel').value;
  if (!model) throw new Error('اختر الموديل أولاً');

  const systemPrompt = buildScriptSystemPrompt(lang, dialect, tone);
  const userPrompt = buildScriptUserPrompt(topic, lang, dialect, tone, research);

  const res = await aiFetch('/chat/completions', {
    method:'POST',
    headers:{'Content-Type':'application/json'},
    body: JSON.stringify({
      model,
      messages:[
        {role:'system', content: systemPrompt},
        {role:'user', content: userPrompt},
      ],
      temperature: 0.7, // أعلى قليلاً للإبداع
    }),
  });
  const data = await res.json();
  return data.choices?.[0]?.message?.content?.trim() || '';
}
```

`buildScriptSystemPrompt` و `buildScriptUserPrompt` يبنيوا تعليمات مفصّلة لكل لهجة:
- **فصحى**: عربية فصحى معاصرة، جمل واضحة وموجزة
- **خليجي**: لهجة خليجية ودودة، مصطلحات محلية
- **مصري**: لهجة مصرية حيوية، روح دعابية خفيفة
- إلخ.

السكربت يُكتب بصيغة منظّمة: كل سطر = فكرة/مشهد محتمل، مع تقسيم واضح (مقدمة/نقاط/خاتمة).

## الخطوة 4: دالة العرض والموافقة — `runScriptGenerator()`

```js
async function runScriptGenerator() {
  // 1. اقرأ الإدخالات
  const topic = document.getElementById('genTopic').value.trim();
  const lang = document.getElementById('genLang').value;
  const dialect = document.getElementById('genDialect').value;
  const tone = document.getElementById('genTone').value;
  // تحقق من العنوان
  if (!topic) { showError('اكتب عنوان/موضوع أولاً'); return; }

  // 2. ابحث (إن أمكن)
  statusEl.textContent = '🔍 جاري البحث على الإنترنت...';
  let research = [];
  try { research = await searchWeb(topic); }
  catch(e){ showWarning('تعذّر البحث، سأعتمد على معرفة النموذج'); }

  // 3. ولّد السكربت
  statusEl.textContent = '✍️ جاري كتابة السكربت...';
  const script = await generateScriptFromAI(topic, lang, dialect, tone, research);

  // 4. اعرض في صندوق المعاينة (قابل للتعديل)
  document.getElementById('scriptPreviewWrap').style.display = 'block';
  document.getElementById('scriptPreview').value = script;
  statusEl.textContent = '✓ السكربت جاهز — راجعه وعدّله ثم اضغط موافق';
}
```

**زر الموافقة** (`#approveScriptBtn`):
- يأخذ نص `#scriptPreview` (بعد تعديل المستخدم إن وجد)
- يملأه في `#scriptInput`
- يستدعي `generateScenes()` الموجودة تلقائياً
- ينتقل للـ Scenes tab أو يبقى في المعاينة

**زر إعادة التوليد** (`#regenScriptBtn`): يعيد استدعاء `runScriptGenerator`.

## الخطوة 5: تحديث `SCENE_SCHEMA_DESC` (الـ prompt الحالي)

الـ prompt الحالي (السطر 864) يذكر **7 أنواع فقط**، لكن التطبيق فيه **25 نوع**. التحديث:

```js
const SCENE_SCHEMA_DESC = `You are a video scene planner...
Scene types and their fields:
- {"type":"intro",...}
- {"type":"title",...}
- {"type":"bullet","fields":{"title":"...","items":[...]}}
- {"type":"stat","fields":{"label":"...","value":"...","unit":"..."}}
- {"type":"comparison","fields":{...}}
- {"type":"quote","fields":{"text":"...","author":"..."}}
- {"type":"outro",...}

New types — choose the MOST FITTING for each piece of content:
- {"type":"checklist","fields":{"title":"...","items":["step 1","step 2",...]}}  ← for lists of steps/tasks
- {"type":"cta","fields":{"title":"call to action","subtitle":"..."}}
- {"type":"bar_chart","fields":{"title":"...","data":["Label:Value","Label:Value"]}}  ← for numeric comparisons/data
- {"type":"donut","fields":{"title":"...","label":"...","value":50}}  ← for percentages
- {"type":"testimonial","fields":{"quote":"...","name":"...","role":"..."}}
- {"type":"faq","fields":{"question":"...","answer":"..."}}
- {"type":"stat","fields":{"label":"...","value":"...","unit":"..."}}  ← for key numbers/stats
- {"type":"countdown","fields":{"title":"...","items":["1...","2..."]}}  ← for top-N lists

Selection rules — pick the type that BEST matches each content piece:
- Numbers/data/comparisons → stat, bar_chart, donut, comparison
- Lists of points/steps → bullet, checklist
- Questions and answers → faq
- Quotes/testimonials → quote, testimonial
- Closing call to action → cta, outro
Always start with intro, end with outro/cta.`;
```

## اعتبارات أمنية وسلوكية

1. **مفتاح Brave** يُخزّن محلياً في المتصفح فقط (localStorage عبر نظام projects الموجود)، لا يُرسل لأي خادم خارجي عدا Brave نفسه.
2. **فشل البحث** لا يوقف التطبيق — يكمل بمعرفة النموذج.
3. **فشل LM Studio** → رسالة خطأ واضحة (الموجودة في `generateScenes`).
4. **التعديل اليدوي** للسكربت في صندوق المعاينة مسموح بالكامل قبل الموافقة.
5. لا تغيير في سلوك التوليد الحالي `generateScenes()` — يبقى يعمل بنفس الطريقة، فقط prompt أذكى وأنواع أوسع.
