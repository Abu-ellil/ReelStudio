# خطة استقبال الـ Feedback والـ Bugs لأداة ReelStudio

## الجزء الأول: الاستراتيجية (نصايح قبل النشر)

### الترتيب الصحيح للخطوات قبل ما تنشر
1. **ثبّت الـ app على GitHub Pages** — الـ URL لسه فيه placeholder (`USERNAME/REPO`). سجّل النطاق، وزوده في README و PUBLISH.md.
2. **أضف رقم إصدار (Version)** داخل التطبيق — عشان لو في بلاغ يبقى معروف النسخة اللي عليه المشكلة.
3. **جهّز 3 قنوات استقبال** (زي ما اخترت): Telegram + WhatsApp + Discord + GitHub Issues.
4. **أضف زر Feedback داخل التطبيق** (تنفيذ الجزء الثاني).
5. **اعمل Issue Templates** على GitHub (Bug / Feature Request) عشان البلاغات تيجي منظمة.

### قنوات الاستقبال المقترحة ودور كل وحدة
| القناة | دورها | الميزة |
|--------|-------|--------|
| **Telegram** | أسرع قناة للمستخدم العادي | بيوصلني فوراً، سهل للعوام |
| **WhatsApp** | لجمهور أوسع | الناس كلها عليه |
| **Discord** | مجتمع ونقاش | بيبني community حوالين الأداة |
| **GitHub Issues** | للبلاغات التقنية المنظمة | تتبّع وترتيب وأولويات |

> ⚠️ المهم: لأن جمهورك **غير تقني**، الاعتماد الأساسي هيكون على **Telegram/WhatsApp** لأنهم الأسهل — GitHub Issues يبقى اختياري للي يقدر يستخدمه.

### أولويات الـ Bugs لما تيجي
- **P0 (Hotfix فوراً)**: الأداة مش بتشتغل خالص / export بيوقف.
- **P1 (أسبوع)**: فيشر في ميزة أساسية لكن التطبيق بيشتغل.
- **P2 (نسخة جاية)**: تحسينات وميزات جديدة.

## الجزء الثاني: التنفيذ — إضافة آلية Feedback داخل app.html

### المخطط العام
أضيف زر **💬 Feedback** في الـ topbar (جنب زرار اللغة)، يفتح **modal** (بُني من الصفر لأن مفيش واحد موجود) فيه:
1. اختيار **نوع البلاغ**: 🐛 Bug / 💡 اقتراح / 💬 رأي عام
2. حقل نص لوصف المشكلة/الاقتراح
3. **معلومات تلقائية** (Browser + OS + التاريخ) — بتساعدك جداً في التشخيص
4. زر **"نسخ النص"** — للمستخدم يلصقه في Discord / GitHub
5. أزرار لكل قناة تفتح اللينك جاهز بنص مُعبّأ:
   - **Telegram**: `https://t.me/share/url?url=...&text=...`
   - **WhatsApp**: `https://wa.me/?text=...`
   - **Discord**: يفتح لينك سيرفرك + ينسخ النص تلقائياً (Discord مفيهوش share URL مباشر)
   - **GitHub Issues**: يفتح صفحة Issues في الريبو

### التعديلات التفصيلية على app.html

#### 1) CSS جديد (يتم إضافته في `<style>`)
- `.modal-overlay` — خلفية معتمة (backdrop) تغطي الشاشة
- `.modal` — الكارت الوسطاني (model على `.panel` style، token واحد: `var(--panel)`, `var(--border)`)
- `.modal-head`, `.modal-body`, `.modal-foot` — الأقسام
- `.fb-type-btn` — أزرار اختيار نوع البلاغ (toggle `.active`)
- `.fb-channel-btn` — أزرار القنوات (كل وحدة بلون: Telegram أزرق، WhatsApp أخضر، Discord بنفسجي، GitHub أسود)
- toast بسيط `.toast` للتأكيد بعد النسخ/الفتح

#### 2) HTML جديد
**زر Feedback** في الـ topbar (بعد السطر 251، جنب `.lang-switch`):
```html
<button class="fb-trigger" id="feedbackBtn" data-i18n="feedbackBtn">💬 Feedback</button>
```

**Modal** (يُضاف بعد `.layout` أي بعد السطر 588):
- overlay + modal box
- 3 أزرار نوع (bug/suggest/general)
- textarea
- صندوق "auto-info" بيظهر المعلومات التلقائية (مقروء فقط)
- أزرار: نسخ النص + Telegram + WhatsApp + Discord + GitHub + إغلاق

#### 3) i18n (في الـ I18N dictionary، السطور 3249-3356)
أضيف المفاتيح دي في **الفرعين en + ar**:
```
feedbackBtn, fbTitle, fbTypeBug, fbTypeSuggest, fbTypeGeneral,
fbPlaceholder, fbCopy, fbCopyDone, fbSendTelegram, fbSendWhatsApp,
fbSendDiscord, fbSendGithub, fbClose, fbAutoInfo, fbTypeLabel, fbSendLabel
```
- الزر والـ modal بيستخدموا `data-i18n` → الترجمة بتحصل تلقائياً مع `applyI18n()`.

#### 4) JavaScript (في الـ `<script>` الرئيسي)
- `openFeedback()` / `closeFeedback()` — فتح/إغلاق الـ modal
- `getAutoInfo()` — بتجمع `navigator.userAgent`، التاريخ، رقم الإصدار
- `buildFeedbackText()` — بتدمج النوع + نص المستخدم + auto-info في رسالة منسّقة
- `copyFeedback()` — نسخ لـ clipboard + toast "تم النسخ"
- `sendTelegram()` / `sendWhatsApp()` — فتح `window.open()` باللينك + النص مُعبّأ (encodeURIComponent)
- `sendDiscord()` — ينسخ النص + يفتح لينك سيرفرك (المستخدم يلصق النص)
- `sendGithub()` — يفتح صفحة Issues (اختياري: ينسخ النص)
- ربط الأحداث (event listeners) في نهاية الـ script

### ما أحتاجه منك (placeholders هتستبدلها)
في كود الـ JS هحط **4 متغيرات** في أول منطقة الـ feedback، واضحة ومعلّقة، تقدر تستبدلها بنفسك:
```js
const FB_LINKS = {
  telegram: 'https://t.me/YOUR_USERNAME',   // ← اسم تليجرامك
  whatsapp: '',                              // ← wa.me/number لو عايز (اختياري)
  discord: 'https://discord.gg/YOUR_CODE',   // ← كود دعوة سيرفرك
  github:  'https://github.com/Abu-ellil/ReelStudio/issues/new', // ← جاهز من الريبو
};
```
> لو سيرفر Discord أو حساب WhatsApp مش موجودين دلوقتي، الزرار هيفضل موجود بس هيرشد المستخدم للقنوات المتاحة. مش هحذف أي قناة.
