## الهدف
إضافة مشهد جديد اسمه `benchmark_hardware` يعرض بيانات الهاردوير المفصّلة من التقرير: اسم البروسيسور، حجم الرام، سرعات الهارد (disk seq/write/read)، عدد الأنوية، عرض نطاق الرام، إلخ. وربطه بزر **Build Benchmark Reel** بحيث يُضاف تلقائياً لكل تقرير.

البيانات دي موجودة فعلاً في الكائن `report` بعد `normalizeReport` (سطور 962-964): `benchmark_scores` و `system_info`. محتاجين بس مشهد جديد يرسمها.

---

## التعديلات (كلها في `app.html`)

### 1. دوال التسجيل (أربعة أماكن)
إضافة `benchmark_hardware` في:
- **`DEFAULT_DURATIONS`** (سطر 699) — قيمة `5.0` ثانية
- **`DEFAULT_FIELDS`** (بعد سطر 716) — قيم sample افتراضية:
  ```
  {cpu_name, cpu_cores, ram_total, disk_seq, disk_write, disk_read, ram_bw, cpu_single, cpu_multi}
  ```
- **`SCENE_TYPE_NAMES`** (سطر 735) — `'Bench: Hardware'`
- **`DISPATCH`** map داخل `drawScene` (سطر 2433) — `benchmark_hardware: sceneBenchmarkHardware`

### 2. CSS badge color (سطر 122)
إضافة `.scene-type-benchmark_hardware{background:rgba(84,180,255,0.15);color:#54b4ff;}` (لون أزرق يميّزه عن باقي مشاهد الـ benchmark).

### 3. `<option>` في الـ `#newSceneType` select (سطر 542)
إضافة `<option value="benchmark_hardware" data-i18n="typeBenchHardware">Benchmark: Hardware</option>` عشان يظهر في قائمة "+ Add scene".

### 4. i18n labels (مكانين)
- **English** (سطر 3240): `typeBenchHardware:'Bench: Hardware'`
- **Arabic** (سطر 3293): `typeBenchHardware:'معيار: الهاردوير'`

### 5. دالة الرسم الجديدة `sceneBenchmarkHardware(p, f)` (بعد `sceneBenchmarkLeaderboard` ~سطر 1838)
مشهد على نفس ستايل `sceneBenchmarkSecondary` (شبكة 3 أعمدة، header متدرّج، stagger animation):
- **Header**: اسم البروسيسور مختصر + "hardware specs"
- **شبكة 3×3 من الإحصائيات**:
  | CPU | CPU CORES | RAM |
  | RAM BANDWIDTH | DISK SEQ | DISK WRITE |
  | DISK READ | CPU SINGLE | CPU MULTI |
- كل خانة: label رمادي صغير + value أبيض بخط mono
- نفس منطق `fadeWindow` و `easeOutCubic` و stagger الموجود

### 6. ربط المشهد بزر Build (سطر 1130-1137)
داخل حلقة `reports.forEach(r => {...})`، بعد مشهد `benchmark_secondary`، إضافة:
```js
const bs = r.benchmark_scores || {};
const cpu = r.system_info?.cpu || {}, ram = r.system_info?.ram || {};
scenes.push({id:makeId(), type:'benchmark_hardware', fields:{
  cpu_name: cpu.name || '', cpu_cores: String(cpu.cores_logical||cpu.cores_physical||''),
  ram_total: String(ram.total_gb||''), ram_bw: String(bs.ram_bandwidth||''),
  disk_seq: String(bs.disk_seq_mbps||''), disk_write: String(bs.disk_write_mbps||''),
  disk_read: String(bs.disk_read_mbps||''), cpu_single: String(bs.cpu_single||''),
  cpu_multi: String(bs.cpu_multi||''),
}, dur:DEFAULT_DURATIONS.benchmark_hardware});
```

### 7. `sceneSpeakableText` (سطر 2528) — اختياري
إضافة فرع `benchmark_hardware` يرجّع نص مختار للـ captions/TTS (زي: "Hardware specs — CPU, RAM, and disk performance").

## النتيجة
لما تضغط **Build Benchmark Reel** على التقرير، هتلاقي لكل report **4 مشاهد** بدل 3:
1. `benchmark_model` (اسم الموديل + GPU + backend)
2. `benchmark_hero` (tokens/sec + TTFT)
3. `benchmark_secondary` (telemetry: GPU util/temp/power, CPU%, RAM)
4. **`benchmark_hardware` (جديد)** ← البروسيسور، الرام، سرعات الهارد، bandwidth

المشهد الجديد هيظهر كمان في قائمة "+ Add Scene" بحيث تقدر تضيفه يدوياً لأي مشروع.
