## ملخّص المشكلة

العناصر (Bullet / dur / 7 / ↑ ↓ ⎘ ✕) تخرج خارج حدود بطاقة المشهد `.scene-card`.

**السبب الجذري** (مؤكَّد بالأرقام):
- `.scene-card-head` (سطر 101) هو صف flex بقيمة افتراضية `flex-wrap:nowrap` وبدون `overflow`/`min-width:0` على الأبناء.
- محتواه الأدنى ≈ 360px (صورة 34 + select بـ `min-width:120px` + "dur" + input 50 + 4 أزرار ≈100 + فجوات 30).
- المساحة المتاحة ≈ 288px فقط (عمود 340px − panel padding 32px − card padding 20px).
- بما أن عناصر flex لا تتقلّص تحت `min-content` (افتراضي `min-width:auto`)، فإنها تفيض خارج يمين البطاقة → ظهور النصوص خارج الحاوية.

**خطأ ثاني من النوع نفسه:** أزرار `.btn-row-tight` ترث `width:100%` من القاعدة العامة `button` (سطر 73)، فيفيض زر "✕ clear override" على الصف.

## الإصلاحات (داخل `AI_Reel_Studio_Unified.html` فقط)

### 1. `.scene-card-head` — السطر 101
إضافة `flex-wrap:wrap` كشبكة أمان حتى يلتف المحتوى بدل أن يفيض:
```css
/* قبل */
.scene-card-head{display:flex;align-items:center;gap:6px;margin-bottom:6px;}
/* بعد */
.scene-card-head{display:flex;align-items:center;gap:6px;flex-wrap:wrap;margin-bottom:6px;}
```

### 2. عنصر `<select>` (نوع المشهد) — السطر 1152 (inline style)
استبدال `width:auto;min-width:120px;` الثابت بـ flex مرن يتقلّص/يتمدّد، فيستوعب الفراغ ويتقلّص عند الضيق:
```
style="flex:1 1 80px;min-width:0;width:auto;padding:4px 6px;font-size:10.5px;margin:0;"
```

### 3. حقل `dur` input — السطر 1156 (inline style)
إضافة `min-width:0;flex:0 0 50px;` حتى لا يمنع تقلّص الصف:
```
style="width:50px;min-width:0;flex:0 0 50px;padding:3px 6px;font-size:11px;"
```

### 4. `.btn-tiny` (الأزرار ↑ ↓ ⎘ ✕) — السطر 88
ترسيخ الأبعاد والتوسيط حتى تكون الأيقونات منتظمة ومتمركزة ولا تتجاوز:
```css
/* قبل */
.btn-tiny{width:auto;padding:3px 7px;font-size:10px;margin:0;border-radius:5px;}
/* بعد */
.btn-tiny{
  width:auto;padding:3px 7px;font-size:10px;margin:0;border-radius:5px;
  line-height:1;display:inline-flex;align-items:center;justify-content:center;
  min-width:22px;height:22px;
}
```

### 5. `.btn-row-tight button` (صف clear override) — السطر 198
إعادة ضبط `width` الموروث (100%) حتى لا يفيض الزر:
```css
/* قبل */
.btn-row-tight button{margin:0;}
/* بعد */
.btn-row-tight button{margin:0;width:auto;flex:0 0 auto;}
```
مع إضافة قاعدة لـ span حتى لا يفيض نص الـ bgLabel الطويل:
```css
.btn-row-tight span{flex:1;min-width:0;overflow:hidden;text-overflow:ellipsis;white-space:nowrap;}
```

## لماذا هذه الإصلاحات (وليس حلول بديلة)
- `flex-wrap:wrap` + `min-width:0` يحلّان الجذر: يسمحان للعناصر بالتقلّص/الالتفاف بدل الفيض، ويبقيان التصميم على صف واحد حين تتّسع المساحة (افتراضي 340px وما فوق)، ويلتفان بأناقة عند تضييق العمود بالسحب.
- لا أضع `overflow:hidden` على `.scene-card` كلها لتجنّب قصّ القائمة المنسدلة (dropdown) الخاصة بـ `<select>`.
- تغييرات دقيقة، لا إعادة هيكلة، وتحافظ على الهوية البصرية الحالية للبطاقة.

## ما لا يُلمس
- باقي الصفوف (`.transport`, `.row`, `.tab-nav`, `.provider-row`, `.report-item .ri-head`) سليمة وتملك `flex-wrap`/`overflow`/`flex` بالفعل — لا حاجة لها.

## التحقّق
بعد التعديل: فتح الملف، التأكّد من أن صف البطاقة يبقى على سطر واحد عند العرض الافتراضي، وأن سحب الفاصل لتضييق العمود الأيسر يُلتف المحتوى بدل أن يخرج، وأن الأزرار ↑↓⎘✕ متمركزة ومنتظمة الحجم.
